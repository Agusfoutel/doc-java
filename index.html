Documentación del Proyecto RedSocial - Iteración 1

Este documento presenta la implementación de la primera iteración del proyecto RedSocial, desarrollado siguiendo los conceptos teóricos vistos en las clases 1-7 del curso de Programación 2.

Conceptos teóricos aplicados

Programación Orientada a Objetos
  • Clases y objetos
  • Encapsulamiento
  • Métodos de acceso (getters/setters)
Estructuras de datos
  • HashMap (O(1) para búsquedas)
  • ArrayList (Lista dinámica)
  • Stack (LIFO para historial)
  • Queue (FIFO para solicitudes)
Manejo de archivos y JSON
  • Lectura y escritura de archivos
  • Serialización/deserialización con Gson
Pruebas unitarias
  • Tests con JUnit
  • Validación de funcionalidades

Estructura del proyecto

Clase Descripción
Cliente Representa a un usuario de la red social
Accion Registra las operaciones realizadas en el sistema
SolicitudSeguimiento Modela una petición de un cliente para seguir a otro
HistorialAcciones Gestiona el registro y deshecho de acciones
Seguimientos Administra la cola de solicitudes de seguimiento
ListaClientes Clase auxiliar para la serialización/deserialización JSON
JSONManager Maneja la lectura y escritura de archivos JSON
RedSocial Clase principal que integra todas las funcionalidades
Main Punto de entrada de la aplicación con la interfaz de usuario
RedSocialTest Pruebas unitarias para validar el funcionamiento

Desarrollo cronológico

El desarrollo siguió un enfoque incremental, comenzando por las entidades básicas y avanzando hacia la integración completa del sistema.
  1. Diseño de clases básicas (Cliente, Accion, SolicitudSeguimiento)
  2. Implementación de estructuras de datos (HistorialAcciones, Seguimientos)
  3. Desarrollo de la clase RedSocial que integra todas las funcionalidades
  4. Implementación de la persistencia con JSONManager
  5. Creación de la interfaz de usuario en Main
  6. Desarrollo de pruebas unitarias

Conclusiones

La implementación de la Iteración 1 del proyecto cumple con todos los requerimientos especificados en la consigna:

  1. Gestión de Clientes: Estructura eficiente (HashMap) para almacenar y buscar clientes.
  2. Historial de Acciones: Sistema para registrar acciones y deshacer la última acción realizada.
  3. Gestión de Solicitudes: Mecanismo de cola para procesar solicitudes en orden.
  4. Persistencia y Pruebas: Carga de datos desde JSON y pruebas unitarias.

El proyecto demuestra la aplicación práctica de los conceptos teóricos vistos en el curso.
