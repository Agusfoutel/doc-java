Documentación del Proyecto RedSocial - Iteración 1

Este documento presenta la implementación de la primera iteración del proyecto RedSocial, desarrollado siguiendo los conceptos teóricos vistos en las clases 1-7 del curso de Programación 2.

Conceptos básicos de Java

Antes de profundizar en el proyecto, explicaremos algunos conceptos fundamentales de Java:

Clases y Objetos

Clase: Es como un plano o plantilla que define atributos (datos) y métodos (funciones).

Objeto: Es una instancia específica de una clase.

Por ejemplo, la clase Cliente es un plano, y cuando creamos un cliente específico como "Pablo" con DNI "40404040", estamos creando un objeto.

Atributos y Métodos

Atributos: Son variables que pertenecen a una clase.

private String dni;
private String nombre;

Métodos: Son funciones que pertenecen a una clase.

public String getNombre() {
    return nombre;
}
Constructor

Es un método especial que se ejecuta cuando se crea un objeto. Tiene el mismo nombre que la clase:

public Cliente(String dni, String nombre, int scoring) {
    this.dni = dni;
    this.nombre = nombre;
    this.scoring = scoring;
    this.siguiendo = new ArrayList<>();
}
Encapsulamiento

Es el principio de ocultar los detalles internos de una clase y proporcionar métodos para acceder a ellos.

Los modificadores de acceso (private, public) y los métodos getter/setter son ejemplos de encapsulamiento.

private String nombre;  // Atributo privado
public String getNombre() {  // Método público para acceder
    return nombre;
}

Conceptos teóricos aplicados

Programación Orientada a Objetos
  • Clases y objetos: El sistema está estructurado en clases que representan entidades como Cliente, Acción, SolicitudSeguimiento.
  • Encapsulamiento: Se utilizan modificadores de acceso (private, public) para ocultar los detalles internos de las clases.
  • Métodos de acceso: Se implementan getters y setters para acceder a los atributos privados.
Estructuras de datos
  • HashMap: Utilizado en la clase RedSocial para almacenar clientes por su DNI, permitiendo búsquedas en tiempo O(1).
  • ArrayList: Utilizado en la clase Cliente para almacenar los nombres de los clientes que sigue.
  • Stack: Implementado en la clase HistorialAcciones para mantener un registro de acciones que pueden ser deshechas (LIFO).
  • Queue: Implementado en la clase Seguimientos para procesar solicitudes de seguimiento en orden de llegada (FIFO).
Detalles de las estructuras de datos
HashMap (Mapa de hash)
  • Es una estructura que almacena pares clave-valor.
  • Permite búsquedas muy rápidas (tiempo constante O(1)) si conocemos la clave.
  • En nuestro proyecto, usamos el DNI del cliente como clave para encontrar rápidamente un cliente.
  • Ejemplo: clientesPorDni.get("40404040") devuelve el cliente con ese DNI instantáneamente.
ArrayList (Lista dinámica)
  • Es una lista que puede crecer o disminuir dinámicamente.
  • Permite añadir, eliminar y acceder a elementos por su posición.
  • En nuestro proyecto, cada Cliente tiene un ArrayList de nombres de clientes a los que sigue.
  • Ejemplo: cliente.getSiguiendo().add("Pablo") añade "Pablo" a la lista de seguidos.
Stack (Pila)
  • Sigue el principio LIFO (Last In, First Out): el último elemento añadido es el primero en salir.
  • Perfecta para implementar funcionalidades de "deshacer".
  • En nuestro proyecto, usamos Stack para almacenar las acciones realizadas.
  • Ejemplo: acciones.push(accion) añade una acción, acciones.pop() extrae la última acción añadida.
Queue (Cola)
  • Sigue el principio FIFO (First In, First Out): el primer elemento añadido es el primero en salir.
  • Ideal para procesar solicitudes en el orden en que fueron recibidas.
  • En nuestro proyecto, usamos Queue para las solicitudes de seguimiento.
  • Ejemplo: solicitudes.offer(solicitud) añade una solicitud, solicitudes.poll() extrae la primera solicitud añadida.
Manejo de archivos y JSON
  • Lectura y escritura de archivos JSON para persistir la información de los clientes.
  • Uso de la biblioteca Gson para la serialización y deserialización de objetos Java a JSON.
  • La clase JSONManager encapsula toda la lógica de manejo de archivos.
Gson gson = new Gson();
ListaClientes listaClientes = gson.fromJson(reader, ListaClientes.class);

Cómo interactúan las clases entre sí

Flujo de registro de cliente
  1. Main recibe los datos del cliente (DNI, nombre, scoring)
  2. Crea un objeto Cliente con estos datos
  3. Llama a RedSocial.agregarCliente(cliente)
  4. RedSocial valida el DNI y verifica que no exista otro cliente con el mismo DNI
  5. Guarda el cliente en su mapa clientesPorDni
  6. Crea una Accion de tipo "Registro" con los detalles del cliente
  7. Llama a HistorialAcciones.registrarAccion(accion) para guardar la acción
  8. HistorialAcciones añade la acción a su pila acciones
  9. Main guarda los clientes actualizados en el archivo JSON
Flujo de seguimiento
  1. Main recibe los DNIs de los clientes
  2. Busca los clientes con RedSocial.buscarClientePorDni()
  3. Llama a RedSocial.seguirCliente(clienteSolicitante, clienteASeguir)
  4. RedSocial crea una SolicitudSeguimiento con los dos clientes
  5. Llama a Seguimientos.agregarSolicitud(solicitud) para añadir la solicitud a la cola
  6. RedSocial crea una Accion de tipo "Solicitud de seguimiento" con los detalles
  7. Llama a HistorialAcciones.registrarAccion(accion) para guardar la acción
  8. Main llama a RedSocial.procesarSolicitudSeguimiento()
  9. RedSocial obtiene la primera solicitud de Seguimientos
  10. Llama a Cliente.seguir(nombre) para actualizar la lista de seguimiento
  11. Crea una Accion de tipo "Seguimiento" y la registra
Flujo de deshacer acción
  1. Main llama a RedSocial.deshacerUltimaAccion()
  2. RedSocial llama a HistorialAcciones.deshacerUltimaAccion()
  3. HistorialAcciones extrae y devuelve la última acción de la pila acciones
  4. RedSocial identifica el tipo de acción y realiza la operación inversa:
    • Si es "Registro": Elimina el cliente del mapa clientesPorDni
    • Si es "Seguimiento": Llama a Cliente.dejarDeSeguir(nombre) para actualizar la lista de seguimiento

Ejemplos prácticos

Ejemplo 1: Registrar un cliente

...
Ingrese una opcion: 1
Ingrese el DNI del cliente: 12345678
Ingrese el nombre del cliente: Laura
Ingrese el scoring del cliente: 90
Cliente registrado correctamente.
¿Qué ocurre internamente?
  1. Se crea un objeto Cliente con DNI "12345678", nombre "Laura" y scoring 90
  2. Se verifica que el DNI sea válido y no exista otro cliente con el mismo DNI
  3. Se añade el cliente al mapa clientesPorDni
  4. Se crea una acción de tipo "Registro" y se añade al historial
  5. Se guarda la lista actualizada de clientes en el archivo JSON
Ejemplo 2: Seguir a otro cliente
--- Menu Red Social ---
4. Seguir a otro cliente
...
Ingrese una opcion: 4
Ingrese el DNI del cliente que quiere solicitar el seguimiento: 12345678
Ingrese el DNI del cliente al que quiere enviar la solicitud de seguimiento: 40404040
Laura ahora sigue a Pablo.
¿Qué ocurre internamente?
  1. Se buscan los clientes con DNI "12345678" (Laura) y "40404040" (Pablo)
  2. Se crea una solicitud de seguimiento de Laura a Pablo
  3. Se añade la solicitud a la cola de solicitudes
  4. Se crea una acción de tipo "Solicitud de seguimiento" y se añade al historial
  5. Se procesa la solicitud: Laura añade "Pablo" a su lista de seguimiento
  6. Se crea una acción de tipo "Seguimiento" y se añade al historial

@Test
public void testSeguirCliente() {
    RedSocial redSocial = new RedSocial();
    Cliente cliente1 = new Cliente("12345678", "Alice", 95);
    Cliente cliente2 = new Cliente("87654321", "Bob", 88);
    redSocial.agregarCliente(cliente1);
    redSocial.agregarCliente(cliente2);
    redSocial.seguirCliente(cliente1, cliente2);
    redSocial.procesarSolicitudSeguimiento();
    assertTrue(cliente1.getSiguiendo().contains("Bob"));
}

Esta prueba valida que un cliente pueda seguir a otro correctamente.
